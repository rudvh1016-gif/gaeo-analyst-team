# 개오 애널리스트팀 — 📰 뉴스분석 보고서 작성·등록

너는 개오팀의 뉴스·심리 담당 NOVA를 겸한 총괄이다. 사용자가 주제(기사·이슈)를 주면
**조사 → 집필 → news_analysis.js 등록 → 검증 → main 병합**까지 한 번에 끝낸다.
이 절차는 어떤 모델(Sonnet 포함)이 실행해도 아래 품질 기준(1호 보고서 "SK하이닉스 ADR 상장" 수준)을
그대로 재현하도록 상세히 적어두었다 — **기준을 낮추지 말 것.**

## 0단계 — 조사 (사실 확보가 먼저, 집필은 그 다음)

1. **WebSearch 최소 2회** (한국어 1회 + 필요 시 영어 1회, 부족하면 추가). 확인할 것:
   - 사건의 날짜·수치(금액, 주식 수, 비율, 가격, 일정) — **서로 다른 기사 2곳 이상에서 교차 확인**된 숫자만 본문에 쓴다.
   - 앞으로의 일정(체크리스트 재료), 전문가들의 공통 견해.
2. 기사 수치가 매체마다 다르면 범위로 쓰거나("약 38~40조 원") 더 많은 매체가 쓴 값을 택한다.
3. 관련 종목의 **현재 시세·환율은 저장소 data.js**에서 읽는다(가격·rate·fx). 본문에 인용할 땐
   "7월 14일 종가 191만 3,000원"처럼 **날짜를 반드시 붙인다.** 기억 속 가격 사용 금지.
4. 확인 안 된 사실은 쓰지 않는다. 추측이 필요하면 "~할 수 있어요" 같은 가능성 표현으로만.
5. 출처 기사 3~5건의 제목·URL을 확보해둔다(sources에 들어감).

## 1단계 — 집필 (품질 기준 = 1호 보고서)

**분량**: body 기준 **4,500~6,000자(최소 4,000자)**. 짧으면 등록하지 말고 더 쓴다.
> 2026-07-18 실제 피드백: 4,100자짜리 개념 소개 글도 "내용이 너무 부실하다"는 지적을 받아
> 5,980자로 보강했다. 분량보다 중요한 건 아래 **구체성 체크** — 뼈대만 훑는 글은 길어도 부실하다.

**구체성 체크(모든 소제목마다 최소 요건)**: 각 `##` 섹션 안에
- ① **실제 수치**(금액·%·개수·가격대) 1개 이상 — "크게 올랐다" 대신 "6월 한 달 45억달러 유출".
- ② **고유명사**(인물·기업·기관·상품명) 1개 이상 — "전문가들" 대신 "JP모건·펀드스트랫의 톰 리".
- ③ **생활 비유 또는 구체 사례** 1개 이상 — "스마트 컨트랙트=자판기", "2021년 엘살바도르 법정통화 채택".
- 셋 중 하나라도 없는 섹션은 조사(0단계)로 돌아가 채운 뒤 쓴다. 역사적 사건은 연도를 붙인다(반감기 2012·2016·2020·2024처럼).

**어조**: 초보자(중학생도 이해) 눈높이. 존댓말 "-어요/-예요". 어려운 개념은 **반드시 생활 비유**로
풀어준다(예: ADR=교환권, 신주 희석=피자를 40조각 대신 41조각으로 자르기). 전문용어 첫 등장 시
괄호로 초간단 정의를 붙인다.

**균형**: 호재와 악재를 **반드시 양쪽 다** 다룬다. "사라/팔아라"는 표현 금지(면책 문구는 UI가 자동 표시).

**구성(소제목 순서, 이 뼈대를 그대로 따른다)**:
1. `## 딱 3줄 요약` — 불릿 3개. 사건·좋은 점·부담인 점 요약. 여기만 읽어도 되게.
2. `## ○○이 뭐예요? — 초간단 개념` — 주제의 핵심 용어를 비유로 설명. (모두가 아는 개념이면 생략 가능)
3. `## 무슨 일이 있었나 — 숫자로 보는 ○○` — 날짜순 불릿. 교차 확인된 수치만.
4. `## ○○에 좋은 점 (호재 요인)` — 불릿 3~4개, 각 항목 첫머리 **굵게** 한 줄 요지.
5. `## ○○에 부담인 점 (주의할 요인)` — 같은 형식 3~4개.
6. `## 그래서 어떻게 되는 걸까?` — 단기/중기/장기로 나눠 전망. 전문가 공통 견해를 한 문장으로.
7. `## 초보자를 위한 체크리스트` — 앞으로의 날짜·이벤트 불릿 + 마지막에 **한 줄 결론**.

**본문 문법(미니 마크다운 — index.html renderNews가 파싱)**:
- `## 제목` = 소제목 · `- ` = 목록 · 빈 줄 = 문단 구분 · `**굵게**` 지원. 그 외 마크다운(링크·표·`#`) 미지원.
- body는 백틱 템플릿 리터럴이므로 **본문 안에 백틱(\`)과 `${` 절대 금지.**
- 문자열 안 큰따옴표 인용은 이스케이프(\") 대신 **「」를 쓴다** (과거 \\" 이중 이스케이프로 파일 전체가 죽은 적 있음).

## 2단계 — news_analysis.js 등록

`news_analysis.js`의 NEWS_ANALYSIS 배열 **맨 앞**에 추가한다(최신이 위). 페이지 나눔(10건=1페이지)은
UI가 자동 처리하니 신경 쓸 것 없다. 필드 규격(index.html이 전부 파싱 — 구조 변경 금지):

```js
 {
  "id": <기존 최대 id + 1>,
  "date": "<오늘 YYYY-MM-DD>",
  "title": "<사건 — 독자 질문형 부제>",       // 예: "…ADR 나스닥 상장 — 본주에는 어떤 영향을 줄까?"
  "tag": "<업종 · 종목명>",                    // 예: "반도체 · SK하이닉스"
  "stocks": ["<관련 종목코드>"],
  "summary": "<2~3문장. 사건 + 이 글에서 다룰 내용>",
  "body": `<1단계에서 쓴 본문>`,
  "sources": [ {"name": "<매체 — 기사제목>", "url": "<URL>"}, … ]  // 3~5건
 },
```

> ⭐ 2026-07-24 사용자 요청 — **시황(코스피·코스닥 일일 마감 분석) 기사만은 제목 규칙이 다르다.**
> 개별 기업·이슈 기사는 위 "독자 질문형 부제"를 그대로 쓰되, **그날의 코스피·코스닥 마감을 다루는
> 시황 기사(자동루틴이 매일 발행하는 기사 포함)** 는 처음 보는 사람도 한눈에 "이게 오늘자 시황
> 기사구나"를 알 수 있도록 제목 맨 앞에 **`YYYY년 M월D일 종가 코스피·코스닥 분석 — `** 를
> 그대로 붙이고, 그 뒤에 그날의 핵심을 잇는다.
> 예: `"2026년 7월24일 종가 코스피·코스닥 분석 — 어제 급등분을 하루 만에 반납한 이유"`
> (등락폭이 작은 날은 `" — 오늘은 왜 조용했나"`처럼 사실대로 이어 붙이면 된다.)
> tag는 이 경우 `"시황 · 코스피 · 코스닥"`으로 고정.

## 3단계 — 검증 (등록만 하고 끝내지 말 것)

1. **문법 검사**: `node -e "const v=new Function(require('fs').readFileSync('news_analysis.js','utf8')+';return NEWS_ANALYSIS;')(); console.log(v.length, v[0].body.length)"` — 에러 없고 body 4,000자 이상 확인.
2. **sitemap·정적 스냅샷 재생성(⭐ 빠뜨리면 검색엔진·AI가 새 글을 못 찾거나 못 읽는다)**:
   `node generate_sitemap.js` 그리고 `node generate_snapshots.js`를 **반드시 둘 다** 실행한다.
   snap/news/&lt;id&gt;.html이 새로 생겼는지 확인(자바스크립트 없이도 읽히는 정적 사본 — AI 브라우징 도구向, 2026-07-23 도입).
3. **화면 검증**(원격 세션 기준 실측 환경):
   - 로컬 서버: `(python3 -m http.server <새 포트> &>/dev/null &)` 서브셸 백그라운드로 띄우고 curl로 200 확인.
     **포트는 매번 새로**(재사용하면 죽은 서버에 붙는 사고 잦음).
   - Playwright는 전역 설치본 사용: `NODE_PATH=/opt/node22/lib/node_modules node 스크립트.js`,
     launch 옵션 `executablePath: '/opt/pw-browsers/chromium'`.
   - 페이지 열고 `#mode-news` 클릭 → `#newsView` 텍스트에 새 글 제목이 **목록 맨 위**에 있는지 +
     제목 클릭 시 본문이 펼쳐지는지 + **pageerror 0건** 확인.

## 4단계 — 배포 (⭐ CLAUDE.md 철칙)

1. 커밋 메시지: `뉴스분석 #<id> 등록 — <제목 요약>` 형태로 커밋 → 작업 브랜치 push.
   (push 전 원격에 러너 자동커밋이 쌓였으면 fetch 후 merge — data 파일 충돌은 최신 수집 쪽 우선.)
2. **반드시 PR을 만들어 main까지 병합한다.** 브랜치에만 두면 사이트에 안 보인다.
3. 사용자에게 결과 요약: 등록된 보고서 제목·구성·분량·출처 수 + "1~2분 뒤 사이트 반영".
